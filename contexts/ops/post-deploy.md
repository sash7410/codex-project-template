# Post-Deploy Verification

This file should describe what to check after deployment.

Do not fill this in from guesses. Generate or update it only after inspecting the current deployment architecture and context.

## Expected Sections

- Target environment
- Deployment artifact/version
- Health checks
- API checks
- UI checks
- Log checks
- Queue/background job checks
- Monitoring/alarm checks
- Rollback triggers
- Sign-off checklist
