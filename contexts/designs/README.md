# Designs

Design docs go here.

Use this folder for proposed or approved designs that are larger than a ticket but more specific than general wiki knowledge.

Examples:

- `ET-001-product-scope-design.md`
- `ET-002-system-architecture-design.md`
- `ET-003-data-model-design.md`

Rules:

- Tickets request work.
- Design docs explain proposed or approved designs.
- Decisions record durable choices.
- Wiki pages store reusable project knowledge.
- Log meaningful design changes in `contexts/log.md`.
