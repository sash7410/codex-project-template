# Tickets

Each ticket should be a focused work packet suitable for one Codex pass or one subagent.

Rules:

- One objective per ticket.
- Include scope and out-of-scope.
- Include exact write scope for implementation tickets.
- Include acceptance criteria.
- Include handoff instructions.
- Do not let multiple subagents edit the same files in parallel.

Use `contexts/tickets/TEMPLATE.md` for new tickets.
